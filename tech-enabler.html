CCID Authentication Service

A standards-based REST API for verifying caller identity using IETF RFC 8224 and the ATIS SHAKEN framework. Enables service providers to assert and validate telephone number authenticity in real-time, reducing robocall fraud across the PSTN.

Operated by TransUnion via the Tech Enabler platform. All credentials and onboarding are managed directly by the TransUnion team.

Authentication

The CCID API supports two authentication methods. Every request must satisfy at least one.

Credentials are issued by TransUnion. Contact the TransUnion Tech Enabler team to obtain an API key or register IP addresses. There is no self-service onboarding.
1
API Key — Query Parameter
Append apiKey=YOUR_API_KEY to every request URL. The key is provisioned by TransUnion and must be kept confidential.
Example: POST /identity?apiKey=YOUR_API_KEY
2
Remote IP Allowlist — CIDR
Requests from pre-registered static IPs or CIDR ranges are automatically authenticated. No apiKey parameter is needed — the source IP is the credential. IPs must be registered during provisioning.
If no valid authentication is provided the API returns ApiKeyRequired (HTTP 400). An invalid key returns ApiKeyInvalid (HTTP 400).
POST /identity

Authentication Service

Submits a caller identity assertion. The service validates the calling party's telephone number against the SHAKEN framework and returns 200 OK with an empty body on success.

Full endpoint: https://authn.ccid.neustar.biz/ccid/authn/v2/identity

Format: tel:+E.164

tostring

Called party's telephone number in TEL URI format.

Format: tel:+E.164

Unique origination identifier for call tracing and audit logging. Should be unique per call session.
